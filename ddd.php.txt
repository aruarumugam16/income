<H3>HTML5 Input</H3>
<P>Let’s take a look at the following markup.</P>
<PRE class="html" name="code">&lt;section class="loginform cf"&gt;
&lt;form name="login" action="index_submit" method="get" accept-charset="utf-8"&gt;
	&lt;ul&gt;
		&lt;li&gt;&lt;label for="usermail"&gt;Email&lt;/label&gt;
		&lt;input type="email" name="usermail" placeholder="yourname@email.com" required&gt;&lt;/li&gt;
		&lt;li&gt;&lt;label for="password"&gt;Password&lt;/label&gt;
		&lt;input type="password" name="password" placeholder="password" required&gt;&lt;/li&gt;
		&lt;li&gt;
		&lt;input type="submit" value="Login"&gt;&lt;/li&gt;
	&lt;/ul&gt;
&lt;/form&gt;
&lt;/section&gt;
</PRE>
<P>If you have been working with HTML forms before, this will look familiar. 
But, you’ll also notice differences there. The inputs have 
<CODE>placeholder</CODE> and <CODE>required</CODE> attributes, which are new 
attributes in HTML5.</P>
<H4>Placeholder</H4>
<P>The <CODE>placeholder</CODE> attribute allows us to give initial text in the 
input that will disappear when it is in a <CODE>focus</CODE> state or being 
filled in. Previously, we used to do it with JavaScript, but now it becomes a 
lot easier with this new attribute.</P>

<H4>The Downsides</H4>
<P>The new form features offered in HTML5 are powerful and easy to implement, 
but they are still lack in some areas. For instance;</P>
<P><STRONG>The <CODE>placeholder</CODE> attribute</STRONG>, is only supported in 
modern browsers — <STRONG>Firefox 3.7+, Safari 4+, Chrome 4+ and Opera 
11+</STRONG>. So, should you need it to work in unsupported browsers, you can 
use <A href="https://github.com/ginader/HTML5-placeholder-polyfill" rel="external">this 
polyfills</A> in conjunction with <A href="http://www.hongkiat.com/blog/html5-loginpage/modernizr.com" 
rel="external">Modernizr</A>.</P>
<P><STRONG>The same thing goes with the <CODE>required</CODE> 
attribute</STRONG>. The error notification cannot be personalized, the error 
will remain <STRONG>“Please fill out this field”</STRONG> rather than 
<STRONG>“Please fill out your First Name”</STRONG>, this attribute support is 
also limited to the latest browsers.</P>
<P>So, using <A href="http://www.w3schools.com/js/js_form_validation.asp" rel="external">JavaScript 
to validate the required field</A> is (so far) a better option.</P>
<PRE class="javascript" name="code">function validateForm()
{
var x=document.forms["login"]["username"].value;
if (x==null || x=="")
  {
  alert("Please fill out the username");
  return false;
  }
}
</PRE>
<H3>Styling the Forms</H3>
<P>Alright, now let’s decorate our login form with CSS. First we will give the 
background a <A href="http://subtlepatterns.com/purty-wood/" rel="external">wood 
pattern</A> in the <CODE>html</CODE> tag.</P>
<PRE class="css" name="code">html {
	background: url('wood_pattern.png');
	font-size: 10pt;
}
</PRE>
<P>Then, we need to remove the default padding and margin in <CODE>ul</CODE> tag 
that wraps the entire <CODE>inputs</CODE> and floats the <CODE>li</CODE> to the 
left, so the inputs will be displayed horizontally, side by side.</P>
<PRE class="css" name="code">.loginform ul {
	padding: 0;
	margin: 0;
}
.loginform li {
	display: inline;
	float: left;
}
</PRE>
<P>Since we float the <CODE>li</CODE>, the parents will collapse, so we need to 
clear things around the parent with the <A href="http://nicolasgallagher.com/micro-clearfix-hack/" 
rel="external">clearfix hack</A>.</P>
<PRE class="css" name="code">label {
	display: block;
	color: #999;
}
.cf:before,
.cf:after {
    content: ""; 
    display: table;
}

.cf:after {
    clear: both;
}
.cf {
    *zoom: 1;
}
:focus {
	outline: 0;
}
</PRE>
<P><A href="http://www.hongkiat.com/blog/a-look-into-css3-negation-not-selector/" 
rel="external">In CSS3 we have the negation selector</A>. So, we will use it to 
target <STRONG>inputs</STRONG> other than the <CODE>submit</CODE> type, which in 
this case<STRONG> will target the email and password input</STRONG>;</P>
<PRE class="css" name="code">.loginform input:not([type=submit]) {
	padding: 5px;
	margin-right: 10px;
	border: 1px solid rgba(0, 0, 0, 0.3);
	border-radius: 3px;
	box-shadow: inset 0px 1px 3px 0px rgba(0, 0, 0, 0.1), 
				0px 1px 0px 0px rgba(250, 250, 250, 0.5) ;
}
</PRE>
<P>Lastly, we will give a little style decoration for the 
<STRONG>Submit</STRONG> button, as follows.</P>
<PRE class="css" name="code">.loginform input[type=submit] {
	border: 1px solid rgba(0, 0, 0, 0.3);
	background: #64c8ef; /* Old browsers */
	background: -moz-linear-gradient(top,  #64c8ef 0%, #00a2e2 100%); /* FF3.6+ */
	background: -webkit-gradient(linear, left top, left bottom, color-stop(0%,#64c8ef), color-stop(100%,#00a2e2)); /* Chrome,Safari4+ */
	background: -webkit-linear-gradient(top,  #64c8ef 0%,#00a2e2 100%); /* Chrome10+,Safari5.1+ */
	background: -o-linear-gradient(top,  #64c8ef 0%,#00a2e2 100%); /* Opera 11.10+ */
	background: -ms-linear-gradient(top,  #64c8ef 0%,#00a2e2 100%); /* IE10+ */
	background: linear-gradient(to bottom,  #64c8ef 0%,#00a2e2 100%); /* W3C */
	filter: progid:DXImageTransform.Microsoft.gradient( startColorstr='#64c8ef', endColorstr='#00a2e2',GradientType=0 ); /* IE6-9 */
	color: #fff;
	padding: 5px 15px;
	margin-right: 0;
	margin-top: 15px;
	border-radius: 3px;
	text-shadow: 1px 1px 0px rgba(0, 0, 0, 0.3);
}
</PRE>
